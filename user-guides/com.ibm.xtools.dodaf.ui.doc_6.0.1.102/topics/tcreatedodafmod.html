Creating a DoDAF model

By using the appropriate conventions and views, you can create models of complex system architectures to comply with the Department of Defense Architectural Framework (DoDAF) version 1.0 specifications that describe operational environments and their components.

For detailed information about DoDAF and IBM® Rational® tools, refer to the White Paper entitled An IBM Rational Approach to DoDAF on the IBM developerWorks® Web site.
  • Enabling DoDAF modeling capabilities
    Before you can develop DoDAF models, you must enable the DoDAF modeling capabilities.
  • Creating a DoDAF modeling project
    You can open, close, save, and delete DoDAF projects and the files that belong to a project, including models. When you create a DoDAF modeling project, you can create both a new model and a set of DoDAF views.
  • Creating DoDAF Operational View products
    You can create the Operational Node Connectivity Description (OV-2) and Operational Information Exchange Matrix (OV-3) products to describe the elements in an operational environment and to analyze the interactions between those elements.
  • Creating DoDAF Systems View products
    You can create operational realizations in the Systems Event Trace Descriptions (SV-10c) product and the Operational Activity to Systems Function Traceability Matrix (SV-5) product to map systems and system functions to operational capabilities and activities. Systems View products describe relationships between systems resources and the Operational Views.
